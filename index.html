  • Style Guide

    A Style Guide serves as an easy reference for refactoring prior projects and a 'jumping-off' codebase for new projects. Always start with brand approved elements and work out from there.

  • Pattern Library

    The Pattern Library is a collection of pre-compiled components made up of the elements from the Style Guide. The purpose is to demonstrate the way these elements come together to form new components.

  • SDLC

    A process for planning, creating, testing, and deploying an application, component, or information system.

  • Email Rules

    A Style Guide and reference point for email templates.
